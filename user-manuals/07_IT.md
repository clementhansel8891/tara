## 1. IT Command Dashboard (`/core/it`)

**Purpose:** High-level overview of the entire IT infrastructure — system health, ticket volume, device counts, and security posture.

### Reading the Dashboard
1. Navigate to **I.T. Command**.
2. The dashboard displays:
   - **Active Devices** — total registered devices in the fleet
   - **Open Tickets** — IT support requests awaiting resolution
   - **System Health Score** — overall infrastructure health (0–100%)
   - **Critical Alerts** — high-priority system or security events
   - **Uptime Summary** — % uptime of key systems (backend, database, frontend)
3. Click any card to navigate to the relevant tool.

### Alert Priority Levels
| Priority | Color | Response Time |
|----------|-------|---------------|
| Critical | 🔴 Red | Immediate — < 1 hour |
| High | 🟠 Orange | Within 4 hours |
| Medium | 🟡 Yellow | Within 24 hours |
| Low | 🟢 Green | Within 72 hours |

## 2. System Health (`/core/it/health`)

**Purpose:** Real-time monitoring of all system services — uptime, response time, error rates, and resource utilization.

### Reading the System Health Panel
1. Navigate to **I.T. Command → System Health**.
2. Each service shows:
   - **Service Name** (e.g., Backend API, PostgreSQL Database, Frontend)
   - **Status**: HEALTHY / DEGRADED / DOWN
   - **Uptime %** — over last 30 days
   - **Avg. Response Time** — milliseconds
   - **Error Rate** — % of requests that failed
3. Click any service card to see the **detailed health history** and event log.

### Responding to a DEGRADED Service
1. Click the degraded service.
2. Review the **Error Log** — what errors are occurring and how frequently.
3. Check the **Resource Panel** — CPU%, Memory%, Disk usage.
4. If within IT's scope to fix:
   - Click **Restart Service** → triggers a service restart.
   - Click **Scale Up** → increases resource allocation.
5. If root cause is unknown:
   - Click **Escalate to Vendor** → creates a FlowGate escalation request.

### Responding to a DOWN Service
1. Click **Mark as Incident** on a DOWN service.
2. Fill in the **Incident Title**, **Severity** (P1/P2/P3), and **Initial Assessment**.
3. Click **Create Incident** → notifies all registered stakeholders.
4. Update the incident log as you work through the issue.
5. Once resolved, click **Resolve Incident** → the system posts a resolution notice to stakeholders.

### Health Check Schedule
Set up automated health checks:
1. Click **Configure Checks** in the WorkQueue.
2. For each service, set:
   - **Check Interval** (e.g., every 60 seconds)
   - **Alert Threshold** (e.g., alert if response time > 2000ms)
   - **Alert Recipients** (IT team email list)
3. Click **Save Configuration**.

## 3. Device Desk (`/core/it/devices`)

**Purpose:** Full IT asset register — every computer, server, printer, phone, and peripheral owned by the company.

### 3.1 Viewing the Device Inventory
1. Navigate to **I.T. Command → Device Desk**.
2. The **Device Table** shows all registered assets with:
   - Device Name / Hostname
   - Device Type (Laptop, Desktop, Server, Phone, Printer, etc.)
   - Assigned User
   - Department
   - Status (Active, In Storage, Decommissioned, Under Repair)
   - Operating System
   - Purchase Date and Warranty Expiry
3. Use the **FilterBar** to filter by Type, Status, Department, or User.

### 3.2 Registering a New Device
1. Click **New Device** in the WorkQueue.
2. Fill in:
   - **Device Name** (e.g., LAPTOP-JDOE-001)
   - **Device Type**: Laptop, Desktop, Server, Mobile Phone, Tablet, Printer, Network Equipment, Other
   - **Serial Number** (from the device chassis)
   - **Make / Model** (e.g., Dell Latitude 5540)
   - **Processor / RAM / Storage** (spec summary)
   - **Operating System** (e.g., Windows 11 Pro)
   - **Purchase Date**
   - **Purchase Price**
   - **Warranty Expiry Date**
   - **Vendor / Supplier** (linked from Procurement's Supplier Master)
   - **Assigned User** (linked to HR People Core)
   - **Department**
   - **Location** (which office/floor/room)
3. Click **Save Device**.

### 3.3 Assigning a Device to a User
1. Open the device record.
2. Click **Assign to User**.
3. Search and select the employee from the HR directory.
4. Set the **Assignment Date**.
5. Click **Confirm Assignment**.
6. An assignment record is created and the employee is notified.

### 3.4 Logging a Device Issue / IT Ticket
1. Open the device record.
2. Click **Log Issue**.
3. Fill in:
   - **Issue Title** (e.g., "Screen flickering — battery drain")
   - **Priority** (Critical, High, Medium, Low)
   - **Description** of the problem
4. Click **Submit Issue** → creates an IT ticket linked to this device.
5. The ticket is routed to the IT team's Workflow Inbox.

### 3.5 Marking a Device as Under Repair
1. Open the device record.
2. Click **Change Status → Under Repair**.
3. Enter the **Repair Vendor** (external repair shop or internal IT).
4. Enter the **Expected Return Date**.
5. Click **Save**.

### 3.6 Decommissioning a Device
When a device reaches end-of-life:
1. Open the device record.
2. Click **Decommission Device**.
3. Enter: **Decommission Date**, **Disposal Method** (e-waste, trade-in, destroyed), **Notes**.
4. Click **Confirm Decommission**.
5. The device is removed from active inventory and archived.

### 3.7 Warranty Expiry Alerts
- Devices with warranties expiring within **90 days** show a 🟡 yellow alert.
- Devices with warranties expiring within **30 days** show a 🔴 red alert.
- Click **Renew Warranty** → opens a PR in Procurement for warranty renewal.

## 4. Topology Map (`/core/it/topology`)

**Purpose:** Visual network and infrastructure map — shows how all systems, servers, and network equipment are connected.

### Reading the Topology Map
1. Navigate to **I.T. Command → Topology Map**.
2. The map displays nodes representing:
   - **Servers** (Backend, Database, Cache, Message Queue)
   - **Network Equipment** (Routers, Switches, Firewalls, Access Points)
   - **User Endpoints** (Workstations, Laptops — summarized by subnet)
   - **Cloud Services** (External APIs, CDN, Cloud Storage)
3. Lines between nodes show **network connections** with bandwidth capacity.
4. Color coding:
   - 🟢 Green = Healthy connection
   - 🟡 Yellow = Degraded performance
   - 🔴 Red = Connection lost

### Clicking on a Node
1. Click any node to open its **Infrastructure Detail Panel**:
   - IP Address and MAC Address
   - Active connections (inbound/outbound)
   - Current load (CPU, Memory)
   - Last health check result
2. Click **View Device Record** to open the full Device Desk entry for this node.
3. Click **Run Ping Test** to check connectivity from the monitoring server.
4. Click **Trace Route** to see the network path to this node.

### Adding a Node to the Topology
1. Click **Add Node** in the WorkQueue.
2. Select **Node Type**: Server, Network Device, Cloud Service, Endpoint.
3. Enter:
   - **Node Name / Hostname**
   - **IP Address**
   - **Location** (datacenter rack, office floor, cloud region)
4. Connect to existing nodes using the visual connector tool.
5. Click **Save Node**.

## 5. Tech Shop (`/core/it/shop`)

**Purpose:** Internal IT hardware and software procurement requests — staff can browse available tech items and request purchases.

### 5.1 Browsing the Tech Catalog
1. Navigate to **I.T. Command → Tech Shop**.
2. The catalog shows available hardware and software with:
   - Item Name and Description
   - Standard Price (based on last procurement price)
   - Lead Time (estimated delivery time)
   - Category (Laptop, Monitor, Peripheral, Software License, etc.)

### 5.2 Requesting a Tech Item
1. Click **Request** on any catalog item.
2. Fill in:
   - **Quantity**
   - **Justification** (why this item is needed)
   - **Urgency** (Normal, Urgent, Emergency)
   - **Preferred Delivery Date**
3. Click **Submit Request** → creates a PR in Procurement's Requisition Queue.

### 5.3 Adding an Item to the Tech Catalog (IT Admin)
1. Click **Add to Catalog** in the WorkQueue.
2. Fill in: **Item Name**, **Category**, **Description**, **Specifications**, **Standard Price**, **Preferred Vendor**.
3. Click **Save Item**.

## 6. Role Registry (`/core/it/roles`)

**Purpose:** Define, manage, and audit all system roles and their permissions across the Zenvix platform.

### 6.1 Viewing the Role Registry
1. Navigate to **I.T. Command → Role Registry**.
2. All defined roles are listed with:
   - **Role Name** (e.g., HR Manager, Finance Staff, Sales Rep)
   - **Department Scope**
   - **Permission Count** — number of actions this role is authorized for
   - **Assigned Users** — how many users currently hold this role

### 6.2 Viewing a Role's Permissions
1. Click any role row.
2. The **Permission Matrix** opens showing every module and action:
   - ✅ = Permitted
   - ❌ = Denied
   - 👁 = View only
3. Review which departments and tools this role has access to.

### 6.3 Creating a New Role (IT Admin)
1. Click **New Role** in the WorkQueue.
2. Enter:
   - **Role Name**
   - **Department** (or cross-departmental)
   - **Description**
3. In the **Permission Matrix**, toggle each permission on/off.
4. Click **Save Role**.

### 6.4 Editing a Role's Permissions
1. Open the role record.
2. Click **Edit Permissions**.
3. Toggle permissions on or off.
4. Click **Save Changes**.
5. All users with this role are immediately affected by the change.

> **⚠ Caution:** Editing permissions on widely-used roles affects all users with that role. Test with a test account before applying broad changes.

## 7. Account Control (`/core/it/accounts`)

**Purpose:** User account management — create accounts, reset passwords, manage 2FA, and lock/unlock accounts.

### 7.1 Viewing All User Accounts
1. Navigate to **I.T. Command → Account Control**.
2. The **Account Table** shows all user accounts with:
   - Email, Full Name, Role, Department
   - Status (Active, Locked, Pending, Deactivated)
   - Last Login timestamp
   - 2FA Status (Enabled/Disabled)

### 7.2 Creating a New User Account
1. Click **New Account** in the WorkQueue.
2. Fill in:
   - **Full Name**
   - **Work Email** (becomes the login username)
   - **Role** (select from Role Registry)
   - **Department**
3. Click **Create Account**.
4. A temporary password is generated and emailed to the new user.
5. The user must change their password on first login.

### 7.3 Resetting a Password
1. Find the user account in the Account Table.
2. Click the user row to open the account detail.
3. Click **Reset Password**.
4. A new temporary password is generated and emailed to the user.

### 7.4 Locking / Unlocking an Account
1. Open the account detail.
2. To lock: Click **Lock Account** → enter the reason (e.g., Suspicious Activity, Employee Resigned).
3. To unlock: Click **Unlock Account**.
4. Locked accounts cannot log in until unlocked by an IT Admin.

### 7.5 Enabling 2-Factor Authentication
1. Open the account detail.
2. Click **Enforce 2FA** → the user is required to set up 2FA on their next login.
3. Or click **Enable for User** → IT Admin sets up 2FA on behalf of the user.

### 7.6 Deactivating an Account (When an Employee Leaves)
1. Open the account detail.
2. Click **Deactivate Account**.
3. Confirm deactivation — the account is immediately locked.
4. All active sessions are terminated.
5. The account is archived (not deleted) for audit purposes.

> **When HR finalizes a termination in People Core, IT should immediately deactivate the associated user account.**

## 8. System Core (`/core/it/admin`)

**Purpose:** Global system configuration — integrations, environment settings, webhook management, and platform administration.

### 8.1 Managing System Integrations
1. Navigate to **I.T. Command → System Core**.
2. In the **Integrations** panel, all connected external services are listed.
3. Click **Add Integration** to connect a new service:
   - Select the **Integration Type**: Payment Gateway, Email Provider, SMS Gateway, ERP, etc.
   - Enter the API credentials (provided by the vendor).
   - Click **Test Connection** → verifies the credentials are working.
   - Click **Save Integration**.

### 8.2 Managing Webhook Configurations
Webhooks allow Zenvix to send real-time event notifications to external systems.
1. In the **Webhooks** panel, click **New Webhook**.
2. Enter:
   - **Endpoint URL** — the URL of the receiving system
   - **Events to Trigger** — select which Zenvix events to send (e.g., NEW_ORDER, INVOICE_PAID)
   - **Secret Key** — for payload signature verification
3. Click **Test Webhook** → sends a sample payload to the endpoint.
4. Click **Save Webhook**.

### 8.3 Viewing System Logs
1. In the **System Logs** panel, select the log category:
   - **Application Logs** — backend errors and warnings
   - **Audit Logs** — user actions across all departments
   - **Security Logs** — login attempts, failed authentication, permission violations
2. Use filters to narrow by **date**, **severity**, or **module**.
3. Click **Export Logs** → download as compressed log file.

### 8.4 Environment Configuration
1. In the **Environment** panel, view current configuration values:
   - Database connection status
   - API rate limits
   - Feature flags (toggle experimental features on/off)
2. Click **Edit Configuration** (requires Owner-level access).
3. Make changes and click **Apply** → changes take effect on next service restart.

## Troubleshooting

| Issue | Solution |
|-------|---------|
| User cannot log in | Check Account Control — account may be locked or deactivated |
| New user not receiving onboarding email | Verify email address is correct; check spam folder |
| System Health shows a service as DEGRADED | Check the error log for that service; consider restarting |
| Device not appearing in inventory | Register it in Device Desk with all required fields |
| Webhook not sending | Verify the endpoint URL is accessible and the secret key matches |
| 2FA code not working | Verify device time is synchronized; reset 2FA and re-enroll |

## Best Practices

- ✅ Deactivate user accounts **same day** an employee is terminated — coordinate with HR.
- ✅ Enforce 2FA for all admin-level roles.
- ✅ Review the Role Registry quarterly — remove permissions that are no longer needed.
- ✅ Run the SLA Sweep on IT tickets weekly to prevent backlogs.
- ✅ Monitor System Health daily — proactively fix degraded services before they go DOWN.
- ✅ Keep device warranty dates current — expired warranties leave company assets exposed.
